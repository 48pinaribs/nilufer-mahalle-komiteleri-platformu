# Katkı Rehberi

Bu projeye katkı sağlamak isteyen herkese teşekkür ederiz.

## Temel İlkeler

- Bu proje açık kaynaklıdır ancak başıboş değildir
- Kod kalitesi, sürdürülebilirlik ve güvenlik önceliklidir
- Her katkı otomatik olarak kabul edilmez

## Katkı Türleri

- Hata bildirimi (bug report)
- Özellik önerisi (feature request)
- Kod katkısı (pull request)
- Dokümantasyon iyileştirmesi

## Issue Açma

- Açıklayıcı başlık kullanın
- Beklenen davranışı net yazın
- Varsa ekran görüntüsü veya örnek ekleyin

## Pull Request Süreci

- PR’lar doğrudan `main` branch’e açılmaz
- Kod standartlarına uygun olmalıdır
- Açıklama içermeyen PR’lar değerlendirilmez
- Core Team onayı olmadan merge edilmez

## Kod Standartları

- Okunabilirlik önceliklidir
- Modüler yapı korunmalıdır
- Gereksiz soyutlamalardan kaçınılır

## İletişim

Sorular ve öneriler için GitHub Issues kullanılmalıdır.
